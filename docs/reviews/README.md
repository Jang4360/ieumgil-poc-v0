# docs/reviews 사용 규칙

## 목적

이 디렉터리는 각 계획 문서 또는 구현 슬라이스의 실행 결과를 남기는 곳이다. 성공 여부와 관계없이 결과를 반드시 `md` 파일로 기록한다.

## 작성 시점

- 구현 성공 기준을 충족했을 때
- 실패 후 원인 수정 재시도를 1회 이상 수행했을 때
- 최대 2회 재시도 후에도 해결 방안이 없을 때

## 기본 규칙

- 각 작업은 성공 기준 미달 시 최대 2회까지 재시도한다.
- 2회 재시도 후에도 해결 방안이 없으면 실패 리뷰를 남기고 중단 또는 에스컬레이션한다.
- 성공한 작업도 반드시 성공 리뷰를 남긴다.
- 리뷰 문서는 작업 단위별로 별도 파일을 만든다.

## 파일명 규칙

- 기본 형식: `YYYY-MM-DD_<plan-id>_<success|failure>_review.md`
- 예시:
  - `2026-04-16_02_success_review.md`
  - `2026-04-16_04_failure_review.md`

## 권장 본문 구조

```md
# 리뷰 제목

## 대상 계획

- plan: `02_data_inventory_and_segment_baseline`

## 결과

- status: `success | failure`
- success criteria met:
- retry count:

## 수행 내용

- 무엇을 구현/검증했는지

## 검증 근거

- 실행 명령, 테스트, fixture, 샘플 시나리오

## 이슈 및 리스크

- 남은 문제 또는 주의점

## 다음 액션

- 후속 작업 또는 보류 판단
```
