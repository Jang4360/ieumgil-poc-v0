## 1. API 목록

| 우선순위 | API | 설명 |
| --- | --- | --- |
| P1 | `POST /routes/search` | 경로 탐색 |
| P1 | `POST /route-logs` | 실제 이동 GPS 로그 수집 |
| P2 | `POST /routes/reroute` | 경로 이탈 재탐색 |

## 2. 경로 탐색

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/routes/search` |
| 인증 | 불필요 |
| 설명 | 출발지와 도착지를 기준으로 최단 경로와 안전 경로를 탐색한다. 경로 구간 정보, 위험도 정보, TTS 안내 문구를 응답에 포함한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Content-Type | Y | `application/json` |

#### Query Parameter

> 없음

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| startPoint | object | Y | 출발지 좌표 |
| startPoint.lat | number | Y | 출발지 위도 |
| startPoint.lng | number | Y | 출발지 경도 |
| endPoint | object | Y | 도착지 좌표 |
| endPoint.lat | number | Y | 도착지 위도 |
| endPoint.lng | number | Y | 도착지 경도 |
| routeOptions | array | N | 요청할 경로 옵션 목록. 없으면 `SAFE`, `SHORTEST` 둘 다 반환 |

#### Request Example

```json
{
  "startPoint": {
    "lat": 35.1796,
    "lng": 129.0756
  },
  "endPoint": {
    "lat": 35.1577,
    "lng": 129.0590
  },
  "routeOptions": ["SAFE", "SHORTEST"]
}
```

### Response

#### Success

- HTTP Status: `200 OK`

#### Data Fields

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| routes | array | N | 탐색된 경로 목록 |
| routes[].routeOption | string | N | 경로 옵션. `SAFE`, `SHORTEST` |
| routes[].title | string | N | 경로 표시명 |
| routes[].distanceMeter | number | N | 총 거리 |
| routes[].estimatedTimeMinute | int | N | 예상 소요 시간 |
| routes[].riskLevel | string | N | 전체 경로 위험도. `LOW`, `MEDIUM`, `HIGH` |
| routes[].segments | array | N | 경로 구간 목록 |
| routes[].segments[].sequence | int | N | 구간 순서 |
| routes[].segments[].geometry | string | N | 구간 선형 좌표 |
| routes[].segments[].distanceMeter | number | N | 구간 거리 |
| routes[].segments[].hasStairs | boolean | N | 계단 여부 |
| routes[].segments[].hasCurbGap | boolean | N | 단차 여부 |
| routes[].segments[].hasCrosswalk | boolean | N | 횡단보도 포함 여부 |
| routes[].segments[].hasSignal | boolean | N | 신호등 여부 |
| routes[].segments[].hasAudioSignal | boolean | N | 음향신호기 여부 |
| routes[].segments[].hasBrailleBlock | boolean | N | 점자블록 여부 |
| routes[].segments[].riskLevel | string | N | 구간 위험도 |
| routes[].segments[].guidanceMessage | string | N | TTS 안내 문구 |

#### Success Example

```json
{
  "success": true,
  "data": {
    "routes": [
      {
        "routeOption": "SAFE",
        "title": "안전 경로",
        "distanceMeter": 950,
        "estimatedTimeMinute": 16,
        "riskLevel": "LOW",
        "segments": [
          {
            "sequence": 1,
            "geometry": "LINESTRING(129.0756 35.1796, 129.0760 35.1799)",
            "distanceMeter": 120,
            "hasStairs": false,
            "hasCurbGap": false,
            "hasCrosswalk": true,
            "hasSignal": true,
            "hasAudioSignal": true,
            "hasBrailleBlock": true,
            "riskLevel": "LOW",
            "guidanceMessage": "전방 횡단보도를 건넌 뒤 직진하세요."
          }
        ]
      }
    ]
  },
  "message": null
}
```

### 비고

- `routeOptions`가 없으면 기본값으로 `["SAFE", "SHORTEST"]`를 적용한다.
- `routeOptions`가 `["SAFE"]`면 안전 경로만 반환한다.
- `routeOptions`가 `["SHORTEST"]`면 최단 경로만 반환한다.
- 경로 상세/구간 조회와 TTS 안내 문구는 별도 API로 분리하지 않고 이 응답의 `segments`에 포함한다.

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | 출발지/도착지 좌표 누락 또는 `routeOptions` 값 오류 |
| 404 | RESOURCE_NOT_FOUND | 리소스를 찾을 수 없습니다. | 탐색 가능한 경로가 없음 |
| 500 | INTERNAL_ERROR | 서버 내부 오류가 발생했습니다. | 경로 탐색 처리 중 서버 예외 |

## 3. 실제 이동 경로 로그 수집

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P1 |
| Method | `POST` |
| URL | `/route-logs` |
| 인증 | 불필요 |
| 설명 | 내비게이션 종료 시점에 사용자가 실제 이동한 GPS 로그를 사용자/기기 식별자 없이 서버에 일괄 업로드한다. 추천 기능 활용은 후순위로 둔다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Content-Type | Y | `application/json` |

#### Query Parameter

> 없음

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| routeOption | string | Y | 사용한 경로 옵션. `SAFE`, `SHORTEST` |
| disabilityType | string | Y | 이동 시점의 장애 유형. `VISUAL`, `MOBILITY` |
| startedAt | string | Y | 내비게이션 시작 시각. ISO-8601 |
| endedAt | string | Y | 내비게이션 종료 시각. ISO-8601 |
| distanceMeter | number | N | 실제 이동 거리 |
| points | array | Y | GPS 좌표 목록 |
| points[].sequence | int | Y | 좌표 순서 |
| points[].lat | number | Y | 위도 |
| points[].lng | number | Y | 경도 |
| points[].recordedAt | string | Y | GPS 기록 시각. ISO-8601 |
| points[].accuracyMeter | number | N | GPS 정확도. 단위 meter |

#### Request Example

```json
{
  "routeOption": "SAFE",
  "disabilityType": "MOBILITY",
  "startedAt": "2026-04-13T09:00:00+09:00",
  "endedAt": "2026-04-13T09:18:30+09:00",
  "distanceMeter": 950.4,
  "points": [
    {
      "sequence": 1,
      "lat": 35.1796,
      "lng": 129.0756,
      "recordedAt": "2026-04-13T09:00:02+09:00",
      "accuracyMeter": 8.5
    },
    {
      "sequence": 2,
      "lat": 35.1799,
      "lng": 129.0760,
      "recordedAt": "2026-04-13T09:00:07+09:00",
      "accuracyMeter": 7.9
    }
  ]
}
```

### Response

#### Success

- HTTP Status: `201 Created`

#### Data Fields

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| routeLogId | Long | N | 저장된 경로 로그 ID |

#### Success Example

```json
{
  "success": true,
  "data": {
    "routeLogId": 1
  },
  "message": null
}
```

### 비고

- GPS 좌표는 내비게이션 진행 중 클라이언트 로컬에 임시 저장한다.
- 서버 업로드는 도착 또는 내비게이션 종료 시점에 한 번 수행한다.
- 사용자가 경로 데이터 수집에 동의하지 않은 경우 클라이언트는 이 API를 호출하지 않는다.
- 사용자 또는 기기 단위 식별자를 저장하지 않는다.
- 수집된 로그는 추후 경로 품질 개선/분석용 원천 데이터로 저장하며, MVP에서는 주변 장소 추천에 사용하지 않는다.

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | 필수값 누락, 좌표값 오류, `routeOption`/`disabilityType` 값 오류 |
| 500 | INTERNAL_ERROR | 서버 내부 오류가 발생했습니다. | 경로 로그 저장 중 서버 예외 |

## 4. 경로 이탈 재탐색

| 항목 | 내용 |
| --- | --- |
| 우선순위 | P2 |
| Method | `POST` |
| URL | `/routes/reroute` |
| 인증 | 불필요 |
| 설명 | 사용자가 기존 안내 경로를 이탈했을 때 현재 위치 기준으로 경로를 재탐색한다. |

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| Content-Type | Y | `application/json` |

#### Query Parameter

> 없음

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| currentPoint | object | Y | 현재 위치 좌표 |
| currentPoint.lat | number | Y | 현재 위도 |
| currentPoint.lng | number | Y | 현재 경도 |
| endPoint | object | Y | 도착지 좌표 |
| endPoint.lat | number | Y | 도착지 위도 |
| endPoint.lng | number | Y | 도착지 경도 |
| routeOption | string | Y | 기존 경로 옵션. `SAFE`, `SHORTEST` |

#### Request Example

```json
{
  "currentPoint": {
    "lat": 35.1796,
    "lng": 129.0756
  },
  "endPoint": {
    "lat": 35.1577,
    "lng": 129.0590
  },
  "routeOption": "SAFE"
}
```

### Response

- HTTP Status: `200 OK`

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| routeOption | string | N | 경로 옵션 |
| distanceMeter | number | N | 총 거리 |
| estimatedTimeMinute | int | N | 예상 소요 시간 |
| riskLevel | string | N | 전체 경로 위험도 |
| segments | array | N | 재탐색된 경로 구간 목록 |

#### Success Example

```json
{
  "success": true,
  "data": {
    "routeOption": "SAFE",
    "distanceMeter": 720,
    "estimatedTimeMinute": 11,
    "riskLevel": "LOW",
    "segments": [
      {
        "sequence": 1,
        "geometry": "LINESTRING(129.0756 35.1796, 129.0760 35.1799)",
        "distanceMeter": 120,
        "hasStairs": false,
        "hasCurbGap": false,
        "hasCrosswalk": true,
        "hasSignal": true,
        "hasAudioSignal": true,
        "hasBrailleBlock": true,
        "riskLevel": "LOW",
        "guidanceMessage": "전방 횡단보도를 건넌 뒤 직진하세요."
      }
    ]
  },
  "message": null
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 400 | INVALID_INPUT | 잘못된 입력입니다. | 현재 위치/도착지 좌표 누락 또는 `routeOption` 값 오류 |
| 404 | RESOURCE_NOT_FOUND | 리소스를 찾을 수 없습니다. | 재탐색 가능한 경로가 없음 |
| 500 | INTERNAL_ERROR | 서버 내부 오류가 발생했습니다. | 경로 재탐색 처리 중 서버 예외 |
